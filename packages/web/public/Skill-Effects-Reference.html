Skill effects reference

Complete field guide for authoring and editing skill definitions.

Resource pools

The four pools that skills can reference as source or destination:

will essence power health
Effect types
stat_modifier

Stat modifier

Adds a flat bonus and/or multiplies a core stat. Scales with rank.

target requiredWhich stat to affect. Options: damage, defense, carry_capacity
addFlat amount added per rank. e.g. add: 2 at rank 3 = +6
multiplyMultiplier. Delta from 1.0 scales with rank. e.g. multiply: 1.1 at rank 3 = ×1.3
conditionOptional. Restrict when this fires (see conditions section below)
{ type: "stat_modifier", target: "damage", add: 3, multiply: 1.1, condition: { weapon_type: "sword" } }
weight_reduction

Weight reduction

Reduces the carry weight of a specific item type (or all items). Flat reduction scales with rank.

target optionalItem type name (e.g. "armor", "potion"). Omit to reduce weight of all items.
addWeight units reduced per rank. e.g. add: 5 at rank 2 = −10 weight
Reductions stack additively across skills. Results are stored by item type key ("all" if no target).
{ type: "weight_reduction", target: "armor", add: 5 }
pool_conversion

Pool conversion

Lets one pool be spent in place of another. Optionally applies a conversion rate.

source requiredThe pool that will actually be spent (substituted in)
destination requiredThe pool that would normally be required
multiplyConversion rate. e.g. multiply: 0.5 means spend half as much from the source pool
addFlat offset applied on top of the rate
Direction: destination is the required pool; source is what's actually spent. Think "pay with source instead of destination."
{ type: "pool_conversion", source: "will", destination: "essence" }
resource_gain

Resource gain

Grants bonus pool resources when the character rests. Only fires during rest actions.

destination requiredWhich pool receives the bonus: will, essence, power, or health
addAmount gained per rank. e.g. add: 4 at rank 2 = +8 on rest
Only triggers when the action context is actionType: "rest". Has no effect during combat or other actions.
{ type: "resource_gain", destination: "power", add: 4 }
utility

Utility

Grants spells, items, or the ability to remove status conditions. Use limit to cap condition removals per day or per rest.

grant_spellName of a spell to add to the character's available spells
grant_itemName of an item to add to granted items
targetName of a status condition this skill can remove (e.g. "poison", "bleed")
limitRequired when using condition removal. Sets how many times it can be used per period
{ type: "utility", target: "poison", limit: { amount: 2, period: "day" } } { type: "utility", grant_spell: "fireball" } { type: "utility", grant_item: "antidote" }
Conditions

All effect types except resource_gain and utility can include a condition block. An effect is skipped if any specified condition field doesn't match the current action context.

weapon_typeOnly fires if the character is using this weapon type (e.g. "sword", "bow")
armor_typeOnly fires if wearing this armor type (e.g. "light", "heavy")
item_typeOnly fires if the relevant item is of this type
is_combatSet true to restrict to combat; false to restrict to non-combat. Omit to always fire.
All specified fields must match. Unspecified fields are ignored (not required to match).
Usage limits

Any effect can include a limit block to cap how many times it fires within a tracking period. Required for condition-removal utility effects.

amountMaximum number of activations before the effect is skipped
period"day" — resets on a new day. "rest" — resets on rest.
Usage is tracked externally via dailyTracker. The engine checks but does not update the tracker — your game layer must record uses.
Rank scaling rules

Additive (add)

Scales linearly: add × rank

add: 5, rank 1 → +5 add: 5, rank 2 → +10 add: 5, rank 3 → +15

Multiplicative (multiply)

Delta from 1.0 scales: 1 + (mult − 1) × rank

mult: 1.1, rank 1 → ×1.10 mult: 1.1, rank 2 → ×1.20 mult: 1.1, rank 3 → ×1.30
Multiple damage or defense modifiers across skills stack: flat bonuses add together, multipliers multiply together.
Full skill definition structure
{ current_rank: 2, effects: [ { type: "stat_modifier", target: "damage", add: 3, multiply: 1.1, condition: { weapon_type: "axe", is_combat: true } }, { type: "resource_gain", destination: "power", add: 5 }, { type: "utility", target: "bleed", limit: { amount: 1, period: "rest" } } ] }
